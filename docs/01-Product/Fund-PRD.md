# Fund 资金管理系统 - 产品需求文档

**版本**: v1.0  
**日期**: 2026-04-03  
**状态**: 待开发

---

## 1. 概述

### 1.1 背景
原 Wallet 菜单功能单一，需要重构为完整的 Fund 资金管理系统，支持多支付方式存款、取款、转账等功能。

### 1.2 目标
- 重构 Wallet → Fund，功能更完整
- 支持多种存款方式（加密货币、银行卡、电汇、信用卡、电子钱包）
- 支持钱包与交易账户之间的资金划转
- 完整的资金流水记录

### 1.3 术语
| 术语 | 说明 |
|------|------|
| Fund Account | 资金账户（钱包） |
| Trading Account | 交易账户（MT5/MT4 账户）|
| Deposit | 存款（外部资金进入平台）|
| Withdraw | 取款（平台资金转出到外部）|
| Transfer | 转账（平台内部账户间划转）|

---

## 2. 账户体系

### 2.1 账户结构
```
用户
├── Fund Account（钱包）
│   ├── 可用余额
│   ├── 冻结金额
│   └── 币种（多币种支持）
│
└── Trading Accounts（交易账户）
    ├── Account #1（USD）
    ├── Account #2（EUR）
    └── Account #3（JPY）
```

### 2.2 资金流动方向
```
[外部资金] → Deposit → [Fund Account] → Transfer → [Trading Account]
                                              ↑
[Trading Account] → Transfer → [Fund Account] → Withdraw → [外部资金]
```

## 3. 功能需求

### 3.1 菜单结构

```
Fund（原 Wallet）
├── 钱包概览（Wallet）
├── 存款（Deposit）
├── 取款（Withdraw）
├── 转账（Transfer）
├── 资金记录（History）
└── 收付款账户（Payment Accounts）
```

### 3.2 钱包概览（Wallet）

#### 3.2.1 功能
- 显示各币种余额
- 显示绑定的交易账户列表
- 快速操作入口（存款/取款/转账）

#### 3.2.2 页面元素
| 元素 | 说明 |
|------|------|
| 总资产 | 所有币种换算为 USD 的总值 |
| 币种列表 | USD/EUR/GBP/JPY 等，显示余额 |
| 交易账户卡片 | 显示账户号、余额、杠杆、快速转账按钮 |
| 最近交易 | 最近 5 笔资金流水 |

### 3.3 存款（Deposit）

#### 3.3.1 存款方式

| 方式 | 子类型 | 说明 |
|------|--------|------|
| **加密货币** | BTC | 比特币转账 |
| | ETH | 以太坊转账 |
| | USDT-ERC20 | ERC20 网络 |
| | USDT-TRC20 | TRC20 网络 |
| | USDC | USDC 转账 |
| **银行卡** | Visa | 信用卡/借记卡 |
| | Mastercard | 信用卡/借记卡 |
| | UnionPay | 银联卡 |
| **电汇** | SWIFT | 国际电汇 |
| | SEPA | 欧元区转账 |
| | Local Bank | 本地银行转账 |
| **信用卡** | Visa | 直接信用卡支付 |
| | Mastercard | 直接信用卡支付 |
| | Amex | 美国运通 |
| **电子钱包** | PayPal | PayPal 转账 |
| | Skrill | Skrill 转账 |
| | Neteller | Neteller 转账 |
| | Alipay | 支付宝 |
| | WeChat Pay | 微信支付 |

#### 3.3.2 存款流程
```
选择存款方式
    ↓
选择/输入存款金额
    ↓
选择目标账户（Fund Account 或 Trading Account）
    ↓
显示支付信息（地址/二维码/跳转第三方支付）
    ↓
用户完成支付
    ↓
系统确认到账 → 更新余额 → 发送通知
```

#### 3.3.3 加密货币存款
- 生成唯一存款地址
- 显示二维码
- 显示网络确认进度（0/3 confirmations）
- 最低存款金额限制

#### 3.3.4 银行卡/信用卡存款
- 3DS 验证支持
- 防欺诈风控检查
- 首次绑卡需小额验证

#### 3.3.5 电汇存款
- 显示银行账户信息
- 生成唯一参考号（Reference Number）
- 上传汇款凭证功能
- 人工确认到账

### 3.4 取款（Withdraw）

#### 3.4.1 取款方式
与存款方式对应，但需先绑定取款账户。

#### 3.4.2 取款流程
```
选择取款方式
    ↓
选择已绑定的取款账户（或添加新账户）
    ↓
输入取款金额
    ↓
选择资金来源（Fund Account 或 Trading Account）
    ↓
安全验证（2FA/邮件验证）
    ↓
提交申请 → 风控审核 → 处理中 → 完成
```

#### 3.4.3 安全要求
- 每日/单笔取款限额
- 新绑定账户 24h 内不可取款
- 大额取款人工审核
- 反洗钱检查

### 3.5 转账（Transfer）

#### 3.5.1 转账类型
| 类型 | 说明 |
|------|------|
| 钱包 → 交易账户 | 从 Fund Account 转入 Trading Account |
| 交易账户 → 钱包 | 从 Trading Account 转出到 Fund Account |
| 交易账户 → 交易账户 | 账户间互转（同币种）|

#### 3.5.2 转账流程
```
选择转出账户
    ↓
选择转入账户
    ↓
输入转账金额
    ↓
确认汇率（如涉及币种转换）
    ↓
确认转账 → 即时到账
```

#### 3.5.3 币种转换
- 不同币种转账时自动换汇
- 显示实时汇率
- 显示预估到账金额

### 3.6 资金记录（History）

#### 3.6.1 筛选条件
- 类型：全部/存款/取款/转账
- 状态：全部/处理中/已完成/失败
- 时间范围：今天/7天/30天/自定义
- 币种：全部/USD/EUR/...

#### 3.6.2 列表字段
| 字段 | 说明 |
|------|------|
| 时间 | 交易时间 |
| 类型 | 存款/取款/转账 |
| 金额 | +1,000.00 USD |
| 状态 | 处理中/已完成/失败 |
| 来源/目标 | 账户信息 |
| 操作 | 查看详情 |

#### 3.6.3 详情页
- 交易哈希（加密货币）
- 参考号
- 手续费
- 交易凭证下载

### 3.7 收付款账户（Payment Accounts）

#### 3.7.1 功能
- 管理取款用的银行账户
- 管理加密货币地址簿
- 管理电子钱包账户

#### 3.7.2 银行账户管理
- 添加银行账户
- 银行名称、账户名、账号、SWIFT/IBAN
- 验证小额打款
- 设为默认
- 删除（需无待处理取款）

#### 3.7.3 加密货币地址簿
- 添加常用地址
- 地址标签（如"个人钱包"）
- 地址验证

---

## 4. 非功能需求

### 4.1 性能
- 转账即时到账（< 3s）
- 余额查询 < 1s
- 历史记录加载 < 2s

### 4.2 安全
- 所有资金操作需 2FA
- 敏感操作邮件通知
- 异常交易风控拦截

### 4.3 费率
| 操作 | 费率 |
|------|------|
| 存款-加密货币 | 免费 |
| 存款-银行卡 | 2.5% |
| 存款-电汇 | 免费（银行费用自理）|
| 取款-加密货币 | 网络费 |
| 取款-银行卡 | 1% + 固定费用 |
| 转账-内部 | 免费 |
| 币种转换 | 0.5% 点差 |

### 5.1 旧路由
```
/portal/wallet
/portal/wallet/deposit
/portal/wallet/withdraw
/portal/wallet/transactions
```

### 5.2 新路由
```
/portal/fund                    → 钱包概览
/portal/fund/deposit           → 存款
/portal/fund/withdraw          → 取款
/portal/fund/transfer          → 转账
/portal/fund/history           → 资金记录
/portal/fund/accounts          → 收付款账户
```

### 5.3 重定向
旧路由 301 重定向到新路由，保持兼容性。
