# KYC 身份认证系统 - 产品需求文档

**版本**: v1.0  
**日期**: 2026-04-03  
**状态**: 待开发

---

## 1. 概述

### 1.1 背景
外汇交易平台合规要求用户完成身份认证（KYC），根据用户居住地区不同，认证流程和所需材料有所差异。

### 1.2 目标
- 支持 9 个地区（越南、泰国、印度、阿拉伯、韩国、日本、法国、西班牙、巴西）的差异化 KYC 流程
- 集成 eKYC 自动审核（Mock 阶段）
- OCR 智能识别证件信息，自动填充表单
- 支持活体识别（眨眼、摇头动作）
- 人工复核机制

### 1.3 术语
| 术语 | 说明 |
|------|------|
| eKYC | 电子身份认证，含 OCR + 活体 + 风险筛查 |
| OCR | 光学字符识别，提取证件文字信息 |
| Liveness | 活体检测，防止照片/视频攻击 |
| PEP | 政治敏感人物（Politically Exposed Person）|
| AML | 反洗钱（Anti-Money Laundering）|

---

## 2. 用户旅程

```
用户进入 KYC 流程
    ↓
[步骤1] 选择证件类型 → 上传证件 → OCR 识别 → 确认/修正信息
    ↓
[步骤2] 活体识别（如地区要求）→ 眨眼 → 摇头 → 人脸比对
    ↓
[步骤3] 填写个人信息 → 教育背景 → 投资经验 → 财务状况 → 各类声明
    ↓
[步骤4] 阅读并签署协议 → 风险披露 → 隐私政策 → 反洗钱声明
    ↓
提交审核 → 系统初审（eKYC）→ 人工复核 → 结果通知
```

### 3.2 步骤1：证件认证

#### 3.2.1 证件上传
- **支持格式**: JPG, PNG, PDF
- **文件大小**: 单张 ≤ 10MB
- **拍摄要求**: 
  - 证件四角完整入镜
  - 文字清晰可读
  - 无反光、无遮挡

#### 3.2.2 OCR 识别字段
| 字段 | 身份证 | 护照 | 驾照 |
|------|--------|------|------|
| 证件号码 | ✅ | ✅ | ✅ |
| 姓名 | ✅ | ✅ | ✅ |
| 出生日期 | ✅ | ✅ | ✅ |
| 性别 | ✅ | ✅ | ✅ |
| 国籍 | ✅ | ✅ | ❌ |
| 证件有效期 | ✅ | ✅ | ✅ |
| 签发机关 | ✅ | ✅ | ❌ |
| 地址 | ✅ | ❌ | ✅ |

#### 3.2.3 用户确认流程
1. OCR 识别完成后展示提取的信息
2. 用户可编辑修正识别错误
3. 确认后进入下一步

### 3.3 步骤2：活体识别

#### 3.3.1 动作要求
1. **眨眼检测**: 用户自然眨眼 1-2 次
2. **摇头检测**: 用户缓慢左右摇头

#### 3.3.2 技术要求
- 检测时间窗口：10 秒
- 失败重试次数：3 次
- 活体分数阈值：≥ 0.85

### 3.4 步骤3：个人信息表单

#### 3.4.1 基础信息（所有地区）
- 姓名（自动填充，可修改）
- 出生日期（自动填充，可修改）
- 性别（自动填充，可修改）
- 国籍（自动填充，可修改）
- 证件号码（自动填充，可修改）
- 证件有效期（自动填充，可修改）
- 居住地址
- 联系电话
- 电子邮箱

#### 3.4.2 教育背景（所有地区）
- 最高学历（高中/本科/硕士/博士/其他）
- 专业领域
- 毕业院校

#### 3.4.3 投资经验（所有地区）
- 交易年限（<1年/1-3年/3-5年/>5年）
- 交易产品（外汇/股票/期货/期权/加密货币/基金）
- 交易频率（每日/每周/每月/偶尔）
- 年均交易量

#### 3.4.4 财务状况（所有地区）
- 年收入范围
- 净资产范围
- 投资目标（保值/增值/投机）
- 风险承受能力（保守/稳健/积极/激进）

#### 3.4.5 声明类（按地区配置）

|----------|----|----|----|----|----|----|----|----|----|
| 美国人声明 | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| PEP 声明 | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| 军队/政治背景 | ✅ | ❌ | ✅ | ✅ | ❌ | ❌ | ❌ | ❌ | ❌ |
| 金融专业声明 | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |

### 3.5 步骤4：协议签署

#### 3.5.1 必签协议
1. **客户协议**（Client Agreement）
2. **风险披露声明**（Risk Disclosure）
3. **隐私政策**（Privacy Policy）
4. **反洗钱声明**（AML Policy）

#### 3.5.2 签署方式
- 勾选确认 + 电子签名（手写签名或输入姓名）
- 时间戳记录

### 3.6 审核流程

#### 3.6.1 自动审核（Mock）
```typescript
// Mock 审核结果
interface MockKYCResult {
  status: "approved" | "rejected" | "manual_review";
  confidence: number;        // 0-1 置信度
  rejectionReason?: string;  // 拒绝原因
  riskFlags: string[];       // 风险标记
}
```

#### 3.6.2 人工复核
- 自动审核置信度 < 0.9 时触发人工复核
- 复核人员在后台查看：证件照片、活体视频、表单信息
- 复核时效：24 小时内

#### 3.6.3 状态通知
- 审核通过：邮件 + App 推送
- 审核拒绝：邮件说明原因，可重新提交
- 需补充材料：邮件说明需补充内容

## 4. 非功能需求

### 4.1 性能
- 页面加载 < 2s
- OCR 识别 < 5s
- 活体检测 < 10s

### 4.2 安全
- 证件照片加密存储
- 传输使用 HTTPS
- 敏感字段脱敏展示

### 4.3 兼容性
- 支持移动端摄像头调用
- 支持桌面端文件上传
