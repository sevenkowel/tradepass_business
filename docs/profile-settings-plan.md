# 个人中心 & 系统设置功能规划

## 功能概述

为 Backoffice 后台管理系统补充用户头像下拉菜单中的两个核心功能页面：
1. **个人中心 (Profile)** - 当前登录用户的个人信息管理
2. **系统设置 (Settings)** - 用户级系统偏好设置

---

## 一、功能清单

### 1.1 个人中心 (Profile)

| 功能模块 | 功能点 | 优先级 | 说明 |
|---------|--------|--------|------|
| **个人信息展示** | 头像、用户名、邮箱、角色 | P0 | 只读展示当前登录用户基本信息 |
| **个人信息编辑** | 修改姓名、手机号、头像 | P0 | 可编辑的个人信息字段 |
| **密码修改** | 当前密码验证 + 新密码设置 | P0 | 安全密码修改流程 |
| **2FA 管理** | 启用/禁用双因素认证 | P1 | 绑定/解绑 2FA 设备 |
| **登录历史** | 最近登录记录查看 | P1 | 显示登录时间、IP、设备信息 |
| **活动日志** | 个人操作审计日志 | P2 | 用户在系统中的关键操作记录 |

### 1.2 系统设置 (Settings)

| 功能模块 | 功能点 | 优先级 | 说明 |
|---------|--------|--------|------|
| **界面设置** | 主题模式（跟随系统/亮色/暗色） | P0 | 用户级主题偏好 |
| **界面设置** | 侧边栏折叠状态记忆 | P0 | 记住用户的侧边栏偏好 |
| **界面设置** | 表格分页大小设置 | P1 | 默认每页显示条数 |
| **通知设置** | 邮件通知开关 | P1 | 各类邮件通知的订阅管理 |
| **通知设置** | 站内通知开关 | P1 | 站内消息提醒设置 |
| **语言设置** | 界面语言切换 | P2 | 多语言支持（如需要） |
| **数据导出** | 个人数据导出 | P2 | GDPR 合规的数据导出功能 |

## 二、需求文档

### 2.1 个人中心 (Profile)

#### 2.1.1 页面结构

```
/backoffice/profile
├── 页面标题：个人中心
├── 个人信息卡片
│   ├── 头像（可点击更换）
│   ├── 用户名（只读）
│   ├── 邮箱（只读）
│   ├── 角色标签（只读）
│   └── 部门信息（只读）
├── 编辑模式表单
│   ├── 姓名输入框
│   ├── 手机号输入框
│   └── 保存/取消按钮
├── 安全设置区域
│   ├── 修改密码入口
│   └── 2FA 状态与管理
└── 活动记录区域
    ├── 登录历史（最近5条）
    └── 查看全部链接
```

#### 2.1.2 用户故事

**US-001**: 作为后台用户，我希望查看我的个人资料，以便确认我的账户信息是否正确。
- 验收标准：
  - 页面加载时显示当前登录用户的头像、用户名、邮箱、角色
  - 信息从 Auth Store 或 API 获取
  - 角色以标签形式展示

**US-002**: 作为后台用户，我希望编辑我的个人信息，以便保持联系信息的准确性。
- 验收标准：
  - 点击"编辑"按钮进入编辑模式
  - 可修改字段：姓名、手机号
  - 保存时进行表单验证
  - 保存成功后显示 Toast 提示

**US-003**: 作为后台用户，我希望修改我的登录密码，以便保障账户安全。
- 验收标准：
  - 需要输入当前密码进行验证
  - 新密码需要符合密码策略（长度、复杂度）
  - 确认密码需要与新密码一致
  - 修改成功后需要重新登录

**US-004**: 作为后台用户，我希望查看我的登录历史，以便发现异常登录行为。
- 验收标准：
  - 显示最近 5 条登录记录
  - 每条记录包含：时间、IP 地址、浏览器/设备、登录结果
  - 提供"查看全部"链接跳转到完整登录历史

#### 2.1.3 数据模型

```typescript
interface UserProfile {
  id: string;
  username: string;
  email: string;
  fullName: string;
  phone?: string;
  avatar?: string;
  role: {
    id: string;
    name: string;
  };
  department?: string;
  twoFactorEnabled: boolean;
  lastLoginAt: string;
  createdAt: string;
}

interface LoginHistory {
  id: string;
  ip: string;
  userAgent: string;
  browser: string;
  os: string;
  location?: string;
  status: 'success' | 'failed';
  failReason?: string;
  createdAt: string;
}
```

### 2.2 系统设置 (Settings)

#### 2.2.1 页面结构

```
/backoffice/settings
├── 页面标题：系统设置
├── 界面设置卡片
│   ├── 主题模式选择（单选按钮组）
│   │   ├── 跟随系统
│   │   ├── 亮色模式
│     │   └── 暗色模式
│   ├── 侧边栏默认状态（开关）
│   └── 表格默认分页大小（下拉选择）
├── 通知设置卡片
│   ├── 邮件通知总开关
│   ├── 细分邮件通知选项
│   │   ├── KYC 审核通知
│   │   ├── 风险告警通知
│   │   ├── 大额出金通知
│   │   └── 系统公告通知
│   └── 站内通知开关
└── 操作按钮
    ├── 保存设置
    └── 恢复默认
```

#### 2.2.2 用户故事

**US-005**: 作为后台用户，我希望自定义界面主题，以便获得舒适的视觉体验。
- 验收标准：
  - 提供三种主题选项：跟随系统、亮色、暗色
  - 切换后立即生效（无需刷新）
  - 设置保存到 localStorage 或用户配置

**US-006**: 作为后台用户，我希望设置侧边栏默认状态，以便根据我的工作习惯定制界面。
- 验收标准：
  - 开关控制侧边栏默认展开/折叠
  - 设置保存后下次登录生效

**US-007**: 作为后台用户，我希望管理我的通知偏好，以便只接收我关心的通知。
- 验收标准：
  - 邮件通知总开关控制所有邮件通知
  - 细分通知类型可单独开关
  - 站内通知可独立开关
  - 设置保存后立即生效

#### 2.2.3 数据模型

```typescript
interface UserSettings {
  // 界面设置
  theme: 'system' | 'light' | 'dark';
  sidebarCollapsed: boolean;
  tablePageSize: 10 | 20 | 50 | 100;
  
  // 通知设置
  notifications: {
    email: {
      enabled: boolean;
      kyc: boolean;
      risk: boolean;
      withdrawal: boolean;
      announcement: boolean;
    };
    inApp: boolean;
  };
}
```

### 3.3 技术要点

1. **状态管理**
   - 使用 Zustand 管理用户设置状态
   - 设置变更时同步更新 localStorage 用于持久化
   - 主题设置需要与现有 ThemeProvider 集成

2. **表单处理**
   - 使用 React Hook Form 进行表单管理
   - 使用 Zod 进行表单验证
   - 密码修改需要当前密码验证

3. **API 模拟**
   - 继续使用 Mock API 模式（与现有代码保持一致）
   - 模拟网络延迟 (500-1000ms)
   - 模拟错误情况

4. **UI/UX**
   - 遵循现有设计系统（使用 backoffice/ui 组件）
   - 支持暗色模式
   - 响应式布局

## 六、风险与依赖

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 主题设置与现有 ThemeProvider 集成复杂 | 中 | 提前调研现有实现，必要时扩展 ThemeProvider |
| 用户设置需要后端 API 支持 | 低 | 先使用 Mock API，后续可无缝切换 |
| 2FA 功能实现复杂 | 中 | Phase 1 先不做 2FA 管理，仅显示状态 |
| 登录历史数据量大 | 低 | 前端分页，只展示最近记录 |
