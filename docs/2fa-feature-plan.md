# 双因素认证 (2FA) 功能需求与实施计划

## 功能概述

为 Backoffice 系统实现完整的 TOTP (Time-based One-Time Password) 双因素认证功能，提升账户安全性。

---

## 功能清单

### 1. 2FA 状态管理
- [ ] 显示当前 2FA 启用状态（未启用/已启用）
- [ ] 显示 2FA 类型（TOTP）
- [ ] 显示最后配置时间

### 2. 启用 2FA 流程
- [ ] 点击"启用"进入配置向导
- [ ] **Step 1 - 验证身份**：输入当前登录密码确认
- [ ] **Step 2 - 生成密钥**：后端生成 TOTP Secret
- [ ] **Step 3 - 展示二维码**：显示 QR Code 供用户扫描
- [ ] **Step 4 - 验证配置**：用户输入 6 位验证码完成绑定
- [ ] **Step 5 - 备份码**：生成 10 个一次性备份码（用于手机丢失时恢复）

### 3. 禁用 2FA 流程
- [ ] 点击"管理"→"禁用 2FA"
- [ ] 验证当前登录密码
- [ ] 输入当前 6 位 TOTP 验证码确认
- [ ] 禁用成功后清除 Secret

### 4. 重新配置 2FA
- [ ] 点击"管理"→"重新配置"
- [ ] 验证身份（密码 + 当前 TOTP）
- [ ] 重新生成 Secret 和二维码
- [ ] 生成新的备份码（旧备份码失效）

### 5. 备份码管理
- [ ] 显示剩余可用备份码数量
- [ ] 查看备份码（需验证身份）
- [ ] 重新生成备份码（旧码全部失效）

### 6. 登录流程集成
- [ ] 密码验证通过后，检查 2FA 状态
- [ ] 如启用 2FA，显示 TOTP 输入界面
- [ ] 支持使用备份码登录
- [ ] 记住设备功能（可选，7-30天）

## 技术实现

### TOTP 算法
- 使用标准 RFC 6238 TOTP 算法
- 时间窗口：30秒
- 验证码长度：6位数字
- 容错窗口：前后各1个时间窗口（共3个窗口）

### 二维码格式
```
otpauth://totp/TradePass:{username}?secret={secret}&issuer=TradePass&algorithm=SHA1&digits=6&period=30
```

### 数据模型
```typescript
interface TwoFactorConfig {
  enabled: boolean;
  secret?: string;           // 加密存储
  backupCodes?: string[];    // 哈希存储
  createdAt?: string;
  lastUsedAt?: string;
}

interface TwoFactorSetupResponse {
  secret: string;            // 明文 secret（仅展示一次）
  qrCodeUrl: string;         // QR Code Data URL
  backupCodes: string[];     // 明文备份码（仅展示一次）
}
```

## API 接口设计

```typescript
// 获取 2FA 状态
GET /api/auth/2fa/status
Response: { enabled: boolean; createdAt?: string; backupCodesRemaining?: number }

// 开始启用 2FA（验证密码后生成 Secret）
POST /api/auth/2fa/setup/init
Body: { password: string }
Response: { secret: string; qrCodeUrl: string }

// 完成启用 2FA（验证 TOTP）
POST /api/auth/2fa/setup/verify
Body: { code: string }
Response: { backupCodes: string[] }

// 禁用 2FA
POST /api/auth/2fa/disable
Body: { password: string; code: string }
Response: { success: boolean }

// 查看备份码
POST /api/auth/2fa/backup-codes/view
Body: { password: string }
Response: { backupCodes: string[] }

// 重新生成备份码
POST /api/auth/2fa/backup-codes/regenerate
Body: { password: string; code: string }
Response: { backupCodes: string[] }

// 登录时验证 2FA
POST /api/auth/login/2fa
Body: { code: string; useBackupCode?: boolean }
Response: { token: string; user: User }
```

## 安全考虑

1. **Secret 加密存储**：数据库中的 TOTP Secret 必须加密
2. **备份码哈希**：备份码使用 bcrypt 等算法哈希存储
3. **防暴力破解**：2FA 验证失败 5 次后锁定 30 分钟
4. **一次性备份码**：每个备份码只能使用一次
5. **重新配置失效**：重新配置 2FA 后旧备份码全部失效
6. **HTTPS 传输**：所有 2FA 相关 API 必须 HTTPS
